# Dashboard UI kit — CloudGuard

An interactive, click-through recreation of the CloudGuard fleet cockpit, built
entirely from this design system's tokens and components. It is a high-fidelity
**recreation** of the shipped Next.js dashboard (see the source repo), not a new
design.

## Run it
Open `index.html`. It loads `../../styles.css` and `../../_ds_bundle.js` (the
compiled component library), then mounts the React views.

> The bundle is generated by the design-system compiler. In a freshly created
> system it may not exist until the first turn completes — if the page is blank,
> the bundle simply hasn't been compiled yet.

## What's interactive
- **Sidebar nav** — Dashboard / Servers / Alerts / Logs are live; Analytics,
  Integrations, Feedback, Help, Settings show the "coming soon" treatment.
- **⌘K command palette** — fuzzy nav + actions (press Cmd/Ctrl+K).
- **Server detail drawer** (the net-new surface) — click any server row for a
  drawer with charts, system identity, host alerts, and "Logs for this host".
- **Servers** — status filter, name search, comfortable/compact density toggle.
- **Logs & Activity** — server/level/module/grep controls, volume histogram, and
  the monospace surface with token highlighting + a live-tail toggle.
- **Alerts** — severity filter and optimistic acknowledge.

## Files
| File | Role |
|---|---|
| `index.html` | App shell wiring, command palette, routing, server-detail drawer |
| `data.js` | Fake fleet data (shapes mirror `lib/api/types.ts`) |
| `icons.jsx` | Lucide-equivalent inline icon set (`window.CGIcon`) |
| `AppShell.jsx` | Sidebar + Topbar |
| `FleetChart.jsx` | Multi-series themed line chart |
| `ServerTable.jsx` | Fleet table (composes StatusBadge + ResourceBar) |
| `DashboardView.jsx` | Overview: KPIs, fleet chart, resource/anomaly panel |
| `ServersView.jsx` | Full table + filters + **ServerDetail** drawer |
| `LogsView.jsx` | Log insights: histogram + mono surface |
| `AlertsView.jsx` | Triage list |

Views compose DS primitives via `window.CloudGuardDesignSystem_a66aa9` and never
re-implement a Button, Card, Badge, StatCard, ResourceBar, or StatusBadge.
